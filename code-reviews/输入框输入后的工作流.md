# ChatView 输入问题后的完整消息流程分析

当用户在 ChatView 输入一个问题并按下回车后，消息会经过以下流程：

---

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              前端 (WebView)                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│  ChatTextArea.tsx  →  useMessageHandlers.ts  →  grpc-client-base.ts        │
│       ↓ Enter键            ↓ handleSendMessage()      ↓ postMessage()      │
└────────────────────────────────────┬────────────────────────────────────────┘

├─────────────────────────────────────────────────────────────────────────────┤
│  grpc-handler.ts  →  newTask.ts / askResponse.ts  →  Controller  →  Task   │

┌─────────────────────────────────────────────────────────────────────────────┐
```

## 🔄 数据流总结

1. **用户输入** → `ChatTextArea` 捕获 Enter 键
2. **消息处理** → `useMessageHandlers.handleSendMessage()` 判断消息类型
3. **RPC 调用** → `TaskServiceClient.newTask()` 或 `askResponse()`
4. **消息传递** → 通过 `postMessage` API 发送到扩展
5. **请求路由** → `grpc-handler.ts` 分发到对应处理器
6. **任务创建** → `Controller.initTask()` → `Task.startTask()`
7. **AI 交互** → Task 内部执行任务循环，调用 AI API
8. **状态同步** → `postStateToWebview()` 将状态推送回前端
9. **UI 更新** → React 重新渲染消息列表

---

## 📝 消息类型说明

### 前端发送的消息类型

- **`newTask`**: 创建新任务（第一条消息）
- **`askResponse`**: 响应 AI 的询问，包含三种响应类型：
  - `yesButtonClicked`: 确认/批准操作
  - `noButtonClicked`: 拒绝操作
  - `messageResponse`: 文本响应

### 后端发送的消息类型

- **`ask`**: 向用户询问（如确认执行命令、请求反馈等）
- **`say`**: 向用户展示信息（如 AI 响应、工具输出等）

---

## 🏗️ 架构设计要点

1. **前后端通信**: 使用类似 gRPC 的 Protocol Buffers 消息格式，通过 VS Code 的 `postMessage` API 进行通信
2. **状态管理**: 后端 Task 类维护任务状态，通过 `postStateToWebview()` 同步到前端
3. **异步等待**: `Task.ask()` 使用 `pWaitFor` 轮询等待用户响应
4. **消息队列**: 消息通过 `messageStateHandler` 管理，支持部分更新（partial message）
